Known Issues & Workarounds
Status

This section is meant to capture real-world quirks and how to deal with them. You can update these entries as you harden your “stable-tested” baselines.

  • GAP: Panel shows “Waiting for network” even though Home Assistant sees it.
    Some builds use an HTTP reachability check to your gateway (for example, 192.168.1.1) to decide when to show “Net: Connected” on the display. If your router blocks these probes or you changed your gateway address, the panel may still be online but show “waiting”. Updating the gateway IP in the config or disabling the HTTP check resolves this.
  • GAP: Auto-close does not trigger while motion is present.
    This is by design. Inside/outside PIR inputs are used to delay or prevent auto-close if motion is detected in the garage area. If your door never auto-closes, check whether a PIR is stuck “active” or aimed at a constantly moving heat source.
  • Ethernet fallback: Panel has Wi-Fi connection but no Ethernet link.
    Panels are designed to prefer Ethernet first and fall back to Wi-Fi. If Ethernet is not plugged in or link is down, this is expected. If Ethernet should be working, verify the LAN8720 wiring, cable, and switch port, then reboot the panel.

When you report a bug, mention which panel you have, which firmware version is installed, and anything you’ve already tried (reboot, cable swap, different port, etc.).
